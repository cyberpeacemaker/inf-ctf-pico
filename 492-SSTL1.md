# Server-Side Template Injection.
In this type of vulnerability, the application takes your input (from the "What do you want to announce" box) and embeds it into a web template (like Jinja2, Mako, or Twig) without properly sanitizing it

# char escape
In bash, parentheses () and curly braces {} have special meanings (like subshells or expansions). When you type {{ config.items() }} without quotes, bash gets confused and throws that syntax error.

# POST-Refirect-GET
When you submit the form, the server processes your payload, stores the result (your "announcement"), and then sends a 302 Redirect to a new page (likely /announce) to show you the result.

By default, curl stops at the redirect. To see the actual flag, you need to tell curl to follow that redirect.

The -L (or --location) flag tells curl to follow any redirects the server sends back.
`curl -L -X POST http://rescued-float.picoctf.net:60203/ --data-urlencode "content={{ cycler.__init__.__globals__.os.popen('cat flag').read() }}"`
